# Query Order Page

## Overview

The `queryOrderPage` interface enables merchants to query multiple orders in a paginated format, helping them retrieve order information based on date ranges, device IDs, and order statuses.


## Interface URL

- UAT: https://uat.test2pay.com/sgs/api/acquire2Query/queryOrderPage
- Production: https://api.payby.com/sgs/api/acquire2Query/queryOrderPage

## Request

### Http Header

<details>
  <summary>Attributes</summary>
  <RequestHeader/>
</details>

### Http Body

- **requestTime**   <font color = ' #7d8793'>Timestamp</font>   <font color = '#f19938'>Required</font>

	- This parameter is used to prevent repeated requests for orders that should have been cancelled due to timeouts.
	- Example value: 1581404947666

- **bizContent**   <font color = ' #7d8793'>Object</font>   <font color = '#f19938'>Required</font>

	Contains the main query parameters for filtering and pagination.

	- **startDate**   <font color = ' #7d8793'>String</font>    <font color = '#f19938'>Required</font>
		- Query start date in YYYY-MM-DD format.
		- Example value: 2022-07-22
		- Maximum length: `10`
	  
	- **endDate**   <font color = ' #7d8793'>String</font>    <font color = '#f19938'>Required</font>
		- Query end date in YYYY-MM-DD format.
		- Example value: 2022-07-22
		- Maximum length: `10`

	- **deviceIdList**   <font color = ' #7d8793'>List\<String\></font>    
		- Filter by specific device IDs. Empty array returns all devices.
		- Example value: []

	- **includeStatusList**   <font color = ' #7d8793'>List\<String\></font>   
		- Filter by order status. Empty array returns all statuses.
		- Possible values: `CREATED`, `PAID_SUCCESS`, `SETTLED`, `FAILURE`
		- Example value: []

	- **sortPageParam**   <font color = ' #7d8793'>Object</font>    <font color = '#f19938'>Required</font>
		- Pagination and sorting configuration.
		
		- **number**   <font color = ' #7d8793'>Int</font>
			- Page number starting from 0.
			- Default value: 0
			- Example value: 0
		
		- **size**   <font color = ' #7d8793'>Int</font>    <font color = '#f19938'>Required</font>
			- Number of records per page.
			- Example value: 20

## Response

### Http Header

**sign**   <font color = ' #7d8793'>String</font>   <font color = '#f19938'>Required</font>

When Botim Money sends response, Botim Money will use its own private key to sign the message, and the merchant uses Botim Money's public key to verify the signature. If the verification is passed, it proves that the response was sent by Botim Money and not faked by others.

### Http Body

**head** 

  <details>
    <summary>Attributes</summary>
    <ResponseHead/>
  </details>

**body** 

**<font color = '#f19938'>Notice </font>**

Body is returned only when `applystatus` = `success`, and `code` = `0`.
If `applystatus` = `error` or `failed`;  or `applystatus` = `success`, `code` !=`0` , that indicates an error. Please check errors and try again.


- **totalElements**   <font color = ' #7d8793'>Int</font>   <font color = '#f19938'>Required</font>
	- Total number of records found in the query.
	- Example value: 6

- **totalPages**   <font color = ' #7d8793'>Int</font>   <font color = '#f19938'>Required</font>
	- Total number of pages available.
	- Example value: 1

- **sortPageParam**   <font color = ' #7d8793'>Object</font>   <font color = '#f19938'>Required</font>
	- Pagination configuration.
	
	- **number**   <font color = ' #7d8793'>Int</font>   <font color = '#f19938'>Required</font>
		- Current page number (0-based).
		- Example value: 0
	
	- **size**   <font color = ' #7d8793'>Int</font>   <font color = '#f19938'>Required</font>
		- Records per page configuration.
		- Example value: 20

- **items**   <font color = ' #7d8793'>List\<Object\></font>   <font color = '#f19938'>Required</font>
	- Array of order records.

	- **merchantOrderNo**   <font color = ' #7d8793'>String</font>   <font color = '#f19938'>Required</font>

		- Maximum length: `64`

	- **orderNo**   <font color = ' #7d8793'>String</font>   <font color = '#f19938'>Required</font>
		- The Botim Money's unique identify number of the order.
		- Example value: 131658483622183687

		- Payment device identifier.
		- Example value: 20001019

	- **status**   <font color = ' #7d8793'>String</font>   <font color = '#f19938'>Required</font>

		- Payment product type.
		- Example value: Basic Payment Gateway

	- **totalAmount**   <font color = ' #7d8793'>Money</font>   <font color = '#f19938'>Required</font>
		- Order total amount with currency.
